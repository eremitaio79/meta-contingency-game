# SPECS

`specs/` e a fonte de verdade funcional do projeto.

## Regra
- crie uma pasta por feature ou capacidade de negocio
- use sempre `requirements.md`, `design.md` e `tasks.md`
- nao crie spec por camada tecnica sem motivo forte
- mova material abandonado para `_archive/`

## Exemplo de nomenclatura
- `specs/authentication/`
- `specs/customer-onboarding/`
- `specs/billing-reconciliation/`
