# CONTRACTS

`contracts/` guarda os acordos tecnicos que implementacao e testes devem respeitar.

## Estrutura
- `api/`: OpenAPI ou equivalente
- `schemas/`: payloads e formatos estruturados
- `events/`: eventos publicados ou consumidos
- `errors/`: catalogo de erros

## Regra
- contrato usado por codigo deve existir aqui
- markdown pode explicar, mas o ideal e manter formato machine-readable quando possivel
